## 🔴 HIGH — resolver logo (causa issues em produção)

### H1. Event listener leak global
**Onde:** `js/01_scene.js:186` — `window.addEventListener('keydown', ...)` registrado com flag `_tBound`, **nunca removido no shutdown**.  
**Risco:** A cada restart de cena (game over → play again), nova instância de listener é adicionada. Após N restarts = N handlers competindo + memory leak.  
**Fix proposto:** Guardar referência da função, remover via `this.events.on('shutdown', () => window.removeEventListener('keydown', handler))`.

### H2. Tutorial flag pollution
**Onde:** `js/17_tutorial.js:571–618` — `_tutConcluir()` limpa flags (`_tutBeamNoDrain`, `_tutBeamNoPull`, `_tutVacasImortais`, `_tutFreezeNave`, `_tutCombustivelCongelado`).  
**Risco:** Se player abandona tutorial mid-step (game over, refresh, scene.restart), flags podem persistir → beam não puxa nada, vacas imortais, nave congelada no jogo normal.  
**Fix proposto:** Reset explícito desses flags no `init()`/`create()` da scene + handler no scene shutdown.

### H3. Tween churn em FX
**Onde:** `js/16_fx.js:88, 145–160` — `_rebuildRain()` e `_rebuildSnow()` fazem `forEach(d => this.tweens.killTweensOf(d))` + recriam pool inteira a cada mudança de count.  
**Risco:** Toggle rápido no debug menu (slider drag) causa stutter. Weather shuffle no PREVIEW também pode disparar várias vezes.  
**Fix proposto:** Debounce de `_rebuildRain/Snow` em 200ms via `Phaser.Math.Easing` ou setTimeout. OU pool reuse — só ajustar quantos estão ativos sem destroy/recreate.

### H4. Fazendeiro `_timer` cleanup race
**Onde:** `js/09_inimigos.js:123–132` — Timer criado em `_spawnFazendeiros()` removido apenas via `_destruirVaca()`.  
**Risco:** Fazendeiro destruído por outro caminho (slam contra rocha em alta velocidade, etc) **não chama `_destruirVaca`** → timer fica orphan. Ao longo de uma sessão longa, dezenas de timers acumulam tentando referenciar fazendeiros mortos.  
**Fix proposto:** Mover `_timer.remove()` pra um `_cleanupFazendeiro(f)` chamado de todo lugar que destrói o farmer (incluindo `_explodir` quando `entity.isEnemy`).

### H5. Filter por frame em `vacas_abduzidas`
**Onde:** `js/01_scene.js:259–265` — Conta cows in beam via `vacas_abduzidas.filter(v => !v.isBurger && !v.isEnemy)` em **toda iteração do update**.  
**Custo:** Com 5 cows = 5 iter/frame × 60 fps = 300 iter/s. Não crítico isoladamente mas escala mal com cap maior.  
**Fix proposto:** Manter contador `this._cowsInBeamCount` atualizado em `_tentarAbduzir` (++) e em `_dropCowsAtCurral` (--). Eliminar filter do update loop.

## 🟡 MEDIUM — cleanup, não urgente

### M1. `Math.sqrt` desnecessário em comparações de distância
**Onde:** `js/07_vacas.js:224, 383`; `js/01_scene.js:323, 337`.  
**Fix:** Usar `dx*dx + dy*dy >= THRESHOLD*THRESHOLD` em vez de `Math.sqrt(dx*dx+dy*dy) >= THRESHOLD`. Já é feito em alguns lugares (inconsistente).

### M2. Graphics objects nunca destroyed
**Onde:** `js/16_fx.js`, `js/18_atmosphere.js`, `js/17_tutorial.js`.  
- `_atmoGfx`, `_atmoFlashGfx` (atmosphere)
- `_tutGfx`, `_tutGlowWorld` (tutorial)
- `fxRain`, `fxSnow` containers

**Risco:** Sobrevivem entre `scene.restart()` se Phaser não fizer cleanup automático.  
**Fix:** Handler `this.events.on('shutdown', () => { ... destroy all ... })`.

### M3. Slot tweens em curral
**Onde:** `js/08_curral.js` — `curral.slots[slotIdx]` contém `{ state, icon, pisca, vaca, slotIdx }`.  
**Risco:** Se curral destruído mid-process, `pisca` tween e `icon` sprite ficam orphan.  
**Fix:** Adicionar null check + destroy handler no curral.

### M4. Sem `scene.shutdown` handler
**Risco:** Cleanup default do Phaser pode perder timers, listeners, graphics customizados em `scene.restart()`.  
**Fix:** `this.events.on('shutdown', () => this._cleanupAll())` que chama todos os cleanup específicos (atmosphere, tutorial, fx, debug menu).

### M5. Balas array não-limitado
**Onde:** `js/09_inimigos.js:52, 60–81` — `this.balas` filtrado por bullets out-of-bounds.  
**Worst case:** 6 torres × ~3s cooldown + ~5 farmers = ~20 bullets/s × ~13s vida = ~260 balas ativas simultâneas.  
**Fix:** Cap rígido (ex: max 100 balas globais) — descartar bala mais antiga ao spawnar nova quando full.

### M6. Dead code (no-ops)
**Onde:** `js/08_curral.js:228–229` — `_coletarDoCurral()` e `_coletarBurgersPerto()` viraram stubs.  
**Fix:** Remover OU adicionar comentário `// DEPRECATED: replaced by _atrairBurgersBeam — keeping no-op for backwards compat` (já tem mas não em todos).

### M7. localStorage debounce
**Onde:** `js/15_debug_menu.js:127` — Cada change de slider chama `_saveDebugCfg()` que escreve no `localStorage`.  
**Custo:** Drag de slider = ~10 writes/segundo. Não-crítico mas wasteful.  
**Fix:** Debounce 500ms ou save só ao fechar menu.

## 🟢 LOW — nice-to-have refactors

### L1. Magic numbers espalhados
- `raioCone` derivado de `40 * 5.55 / 2` (07_vacas.js)
- Distance thresholds: 240 (flee), 220 (cursor reach), 110 (drop curral), 60 (pickup)
- Slot values: `[100, 150, 220]` / fuel `[22, 28, 36]` (08_curral.js)
- Speed thresholds: 0.5, 0.08, 0.05 (inconsistentes entre files)

**Fix:** Centralizar em `js/00_constants.js` ou block top de cada file.

### L2. Duplicate filter logic
**Onde:** `!v.isBurger && !v.isEnemy && !v._inCurral` repetido 4+ vezes em `_tentarAbduzir`, `_dropCowsAtCurral`, `_atualizarIAVacas`, `_tutVacasVivas`.  
**Fix:** Helper `_isAbducibleCow(v)` no prototype.

### L3. Speed thresholds inconsistentes
**Onde:** Vários files usam thresholds diferentes pra "movendo vs parado". Padronizar.

### L4. Preload sem error handling
**Onde:** `js/02_preload.js`.  
**Risco:** Asset 404 falha silencioso (textura missing vira retângulo verde Phaser default).  
**Fix:** `this.load.on('loaderror', file => console.warn('Missing:', file.src))`.

### L5. Mobile dual-input untestado
**Onde:** Branches em `this.isMobile` mas sem teste pra hybrid (web em tablet com trackpad/mouse + touch). Provavelmente quebra parcialmente.

### L6. Tutorial state como FSM explícita
**Onde:** Flags `_tutBeamNoDrain`, `_tutBeamNoPull`, `_tutVacasImortais`, `_tutGravitonDrain2x`, etc. são scattered global state.  
**Risco testabilidade:** Hard to reason about combinações válidas.  
**Fix:** Refator pra `tutorialState` object com `mode` enum (NONE / SAFE_BEAM / FREE_PLAY / DAMAGE_DEMO / ENEMY_LESSON) que define o conjunto de flags.
